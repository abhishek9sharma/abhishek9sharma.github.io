Self-attention: letting tokens look at each other

This is the mechanism that made Transformers work. Each token asks a question (Query), every token advertises what it has (Key), and the matches decide how much each token's content (Value) flows into the result. The reflexive pronoun "itself" resolves directly back to "dog" — both are in the same sentence.

01The Intuition

Query, Key, Value — think Netflix

Imagine you open Netflix in a particular mood. You want something tense, psychological, with a twist ending. That mood is your Query — what you're looking for.

Every title on the platform has tags: "psychological thriller · dark · non-linear". Those tags are the Keys — what each title advertises about itself.

Netflix scores your mood against every title's tags. High match → high score → that show gets recommended more heavily. What you actually receive — the episodes, the acting, the story — is the Value. The Key was just the index; the Value is the content.

Q Query — what this token is looking for
e.g. "itself" looking for the noun it refers to K Key — what this token advertises about itself
e.g. "dog" advertising: animate, noun, subject V Value — the actual content that flows when matched
e.g. the meaning payload "itself" receives from "dog"

Score (Q·K) decides how much to attend. Value is what actually flows. You never receive the Key — it's just the index.

In the model: Q, K, and V are each the token's vector multiplied by a learned weight matrix (W_Q, W_K, W_V). In the example below they're hand-set so the numbers are readable — in a real model they're learned from data, exactly like the embeddings in Step 1.

Step-by-step: one token's attention

Why divide by √dk? Dot products grow with vector dimension. Without scaling, large scores push softmax into near-one-hot outputs with tiny gradients, and training stalls. Dividing by the square root of the key dimension keeps scores in a sane range.
03The Four Stages

Score → Scale → Soften → Blend

Every attention output runs through exactly four operations. You already saw them in the step-by-step calc above — here they are named and explained:

1
Score Q · K
Dot-product of the query with every key. A high score means "this token is relevant to my question." Scores are unbounded — any positive or negative number.
↓
2
Scale ÷ √dk
Divide by the square root of the key dimension. Without this, larger embedding sizes produce bigger dot products, pushing softmax into near-zero gradients and making training stall.
why square root specifically?

If each element of Q and K is drawn from a distribution with variance 1, their dot product has variance dk. Dividing by √dk brings the variance back to 1 regardless of dimension size.

↓
3
Soften softmax
Converts raw scores into weights that sum to 1 — a probability distribution over which tokens to attend to. Higher scores get exponentially more weight; low scores shrink toward zero.
↓
4
Blend × V
Weighted average of all value vectors. Tokens with high attention weight contribute more of their content. The result is a single context-aware vector for this query token.
04The Whole Thing

One formula, four stages

The four stages collapse into a single expression you'll see in every Transformer paper:

Attention(Q,K,V) = softmax( QKT / √dk ) V
QKT = score every key against every query · /√dk = scale · softmax = soften into weights · × V = blend values
This is self-attention because Q, K, and V all come from the same sequence — tokens attend to each other.
cross-attention (used in translation models)

In encoder-decoder models like the original Transformer, the decoder draws Q from its own tokens but K and V from the encoder's output. The math is identical — only the source of K/V changes.

Where this is heading

One attention pattern only captures one kind of relationship. Real models run many attention heads in parallel — one might track subjects, another track position, another track tone — then combine them. That, plus the feed-forward layers and residual connections that wrap around attention, is the full Transformer block. That's Step 4.
