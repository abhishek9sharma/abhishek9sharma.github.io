Intuition-first LLMs · Step 4 of 8

Many heads, then the full Transformer block

One attention pattern sees one kind of relationship. Real models run several in parallel — different heads specialise — then wrap attention in feed-forward layers, residual "shortcuts," and normalization. Stack that block N times and you have the engine of an LLM.

Story so far ▸ a single attention pattern resolving one relationship (Step 3). This step ▸ running many heads at once and wrapping them into the full block you stack N times.
01Multi-Head Attention

Different heads, different jobs

Instead of one set of Q/K/V projections, the model learns several. Each "head" attends to the sentence differently — and because nobody tells them what to specialise in, they divide the labour on their own during training. Here are three heads on "the dog chased it", each with a hand-set pattern to show the idea:

Each head produces its own context vector for every token. The model concatenates all heads' outputs and passes them through one more linear layer (W_O) to mix them back together.

Why this helps. Head 1 can resolve pronouns while Head 2 tracks word order and Head 3 watches sentence-level tone — all at once, in a single layer. GPT-2 small uses 12 heads per layer; GPT-3 uses 96. More heads = more relationship types captured simultaneously.
02The Transformer Block

What wraps around attention

Attention alone isn't enough. A full block adds a feed-forward network (per-token thinking), residual connections (shortcuts that add the input back), and layer normalization (keeps numbers stable). Step through it:

Why residual connections matter ▾

A residual connection computes output = layer(x) + x instead of just layer(x). The + x is a "highway": gradients can flow straight back through it during training, so even very deep stacks (GPT-3 has 96 blocks) keep learning. Without residuals, deep networks suffer vanishing gradients and barely train. It's arguably as important as attention itself.

What the feed-forward network does ▾

After attention mixes information between tokens, the FFN processes each token independently — two linear layers with a non-linearity (Linear → GELU → Linear, where GELU is just a smooth activation function that lets the network learn non-straight-line patterns), usually expanding to 4× the width and back. Intuition: attention gathers the relevant context, then the FFN "thinks about" what it gathered. It holds most of the model's parameters.

03Depth

Stack the block N times

The output of one block is the input to the next — same shape, so they chain cleanly. Early layers learn surface patterns (syntax, nearby words); deeper layers build abstract meaning. Slide to see how model size scales with depth:

Where this is heading

You now have the entire forward pass: tokens → embeddings + position → N transformer blocks → prediction head → next-token probabilities. But every weight started as random noise. How does it learn to produce good predictions? That's training — loss, gradients, and watching embeddings organise themselves out of pure noise. Step 5.
