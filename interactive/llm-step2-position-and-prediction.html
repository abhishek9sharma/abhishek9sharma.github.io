01The Problem

Vectors are orderless bags — by default

A set of embedding vectors carries no notion of order. Without extra information, the model sees the same thing for "dog bites man" and "man bites dog." Click any token below to highlight it, and notice: the raw embedding values are identical regardless of position.

What is a dimension?

Before we show the problem, let's be concrete about what a vector is. Each word is stored as a short list of numbers — one number per dimension. Think of each dimension as a question the model can answer about the word. Here we use 5 readable dimensions:

dimquestion it answersdogman
0 livingis it a living thing?+0.9+0.9
1 humanis it a person?−0.8+0.9
2 gender+male / −female0.0+0.9
3 sizelarge vs small+0.3+0.5
4 actiondoing something?+0.2+0.0

So dog = [0.9, −0.8, 0.0, 0.3, 0.2] and man = [0.9, 0.9, 0.9, 0.5, 0.0]. A vector is just that list. A dimension is just one slot in it.

02Positional Encoding

The fix: stamp each token with its position

Above we saw that dog bites man and man bites dog look identical to the model because the embeddings carry no order. The fix is to add a small extra vector to each embedding that encodes where that token sits. The original Transformer uses sine and cosine waves at different frequencies so every position gets a unique, continuous fingerprint.

"dog bites man" — each token stamped with its position

The mini-chart inside each chip is a sine wave for the chosen dimension across all positions. The dot marks exactly where this token sits on the wave. Notice: same dimension, but each token lands on a different point — that is the unique fingerprint.

Formula:   PE(p, 2d) = sin(p / 100002d/D)   PE(p, 2d+1) = cos(p / 100002d/D)
Low-freq dimensions change slowly (global location). High-freq ones flip fast (local location). Every position gets a unique combination across all dimensions.

"dog" in both sentences — embedding identical, PE different

The word dog sits at position 0 in dog bites man but at position 2 in man bites dog. Its raw embedding is exactly the same both times — only the positional encoding differs. That difference propagates into the combined vector the model sees.

embedding (same both sides) pos encoding (differs) combined = emb + PE
RoPE and learned positions. Modern models like LLaMA and GPT-4 use Rotary Position Embeddings (RoPE) or learned position tables instead of fixed sinusoids, but the idea is the same: inject a position signal so "dog bites man" and "man bites dog" are no longer identical to the model.
03Prediction Head

From a vector back to a word

After the model's layers run (attention, feed-forward — that's Step 3), the output is still a vector. The prediction head multiplies it by the unembedding matrix (the transpose of the embedding table) to get one logit score per vocabulary word. Softmax turns those scores into a probability distribution — it makes every score positive (by exponentiating) and rescales them to sum to 1, so they read as percentages.

Temperature: controlling how "sharp" the distribution is

Before softmax, logits are divided by a temperature T. Low T → the model is very confident (nearly always picks the top word). High T → the distribution flattens, more variety.

1.0

Try T=0.2 (almost always "throne") vs T=3.0 (spread across many words). Real GPT-4 default is around 0.7–1.0.

Sampling strategies: how the model picks a word

Always pick the highest-probability token. Fast and deterministic, but can get repetitive — the model gets "stuck" on high-probability patterns.

Keep only the top-k candidates, then sample from them proportionally. Adds variety while avoiding low-probability noise. GPT-2 used k=40.

Sort by probability, then include tokens until their cumulative probability exceeds p. This automatically adjusts k — when the model is confident, very few tokens are included; when uncertain, more. Nucleus sampling (Holtzman et al., 2020).

The training signal. During training, the model sees the correct next token and computes cross-entropy loss between the predicted distribution and a one-hot vector for that token. Backprop then nudges every weight — the embedding table, the position encodings, and all the layer weights — to increase the probability of the correct answer. That's the whole learning loop; Steps 1–2 are its input and output pipeline.

Where this is heading

You can now feed a sequence into the model and read its next-token prediction. What happens in between? The attention mechanism — which lets token[3] look at token[0] and decide how much it matters. That's Step 3, and it's where the real magic lives.
