01Tokenization

Cutting text into pieces

A model can't read a sentence as one blob — it needs a fixed list of building blocks called a vocabulary. Tokenization is the rule for chopping text into those blocks. There are three common granularities; flip between them and watch the same sentence split differently.

Show the merge rules being learned ▾

BPE starts with single characters, then repeatedly fuses the most frequent adjacent pair into a new token. Each line below is one learned merge, in order. These exact rules are then applied to your text above. · marks a word boundary.

Why subword wins. Characters give a tiny vocabulary but very long sequences. Words give short sequences but a huge, brittle vocabulary that breaks on anything unseen (Kingdoms → unknown). BPE is the compromise real models use: common chunks like king become single tokens, while rare words fall back to smaller pieces — so nothing is ever truly out-of-vocabulary.

Every token gets a number

03Embeddings

Numbers that finally carry meaning

An ID is meaningless, so the model keeps an embedding table: one learned vector per token. During training these vectors arrange themselves so that words used in similar ways end up close together. Below, instead of random training, I've hand-labelled 7 dimensions so you can read them directly — a real model has hundreds of dimensions and nobody hand-labels them, but the principle is identical.

What is a vector? What is a dimension?

A vector is just a list of numbers — like coordinates. A dimension is one slot in that list. We've given each word 7 dimensions, and labelled each one so you can actually read what it means:

dim index
what it measures (hand-labelled for this lesson)
dim 0
living — is this a living thing? (+1 yes, −1 no)
dim 1
human — is it a person? (+1 yes, −1 no)
dim 2
gender — +1 male, −1 female, 0 neutral
dim 3
royalty — is there rank/title involved?
dim 4
size — large (+1) to small (−1)
dim 5
action — does it describe doing something?
dim 6
abstract — concept vs. concrete thing

What is "cosine similarity"? ▾

Measures the angle between two vectors — length is ignored:

cos(θ) = (A · B) / (‖A‖ · ‖B‖)
+1same direction — very similar meaning  0perpendicular — unrelated −1opposite direction — opposite meaning

Why angle, not distance? Two vectors can point the same direction regardless of how long they are — direction is what carries meaning and stays consistent as embedding dimensions grow.

The embedding space, on two axes you choose

Vector arithmetic: the famous analogy

Because meaning lives in directions, you can do algebra on words. Take king, subtract its "maleness," add "femaleness," and you land next to queen.
