Intuition-first LLMs · Step 5 of 8

Training: turning noise into knowledge

Every weight in the model starts as random noise. Training is a loop: predict, measure how wrong you were (loss), and nudge every weight to be slightly less wrong (gradient descent). Below, you'll actually train embeddings live and watch structure emerge from randomness — the payoff promised back in Step 1.

Story so far ▸ the complete model architecture, Steps 1–4. This step ▸ how its randomly-initialised weights actually learn to make good predictions.
01The Loss

Measuring "how wrong" with cross-entropy

The model outputs a probability for every possible next word. The loss compares that distribution to the true next word. If the model gave the correct word high probability, loss is low; if it was confidently wrong, loss is high. Pick the true word and see:

Context: "the king sat on the ___"

—
The formula: loss = −log( P(correct word) ). If the model gave the right word 90% probability, loss = −log(0.9) = 0.10 (tiny). If it gave 2%, loss = −log(0.02) = 3.9 (large). The −log punishes confident mistakes harshly — which is exactly what we want.
02The Update

Gradient descent: roll downhill

The loss is a function of the weights. Its gradient points in the direction of steepest increase — so we step the opposite way to reduce it. The learning rate sets the step size: too small and it crawls, too big and it overshoots. Drag the ball, set a rate, and watch it descend:

Loss landscape (1 weight, for illustration)

0.30

Real models have billions of weights, not one — so the "landscape" is billion-dimensional and impossible to draw. But the rule is identical for every single weight: compute the gradient, step downhill. Backpropagation is just the chain rule applied to compute all those gradients efficiently in one backward pass.
03Watch It Learn

Train real embeddings from noise

This trains a tiny model on a few toy sentences. Each word's embedding starts random. The model learns to predict the next word; as loss drops, words that share contexts drift together. Hit Train and watch the loss curve fall and the word map self-organise — no positions are hand-set this time. It's all learned.

04The Catch

Overfitting: memorising vs. learning

Train too long on too little data and the model memorises the training set instead of learning general patterns. Training loss keeps dropping, but loss on held-out data starts rising — the model is getting worse at the only thing that matters. Watch the gap open up:

Train loss vs. validation loss

The vertical line marks where validation loss bottoms out — the sweet spot to stop training ("early stopping"). Past it, the model overfits. Real training uses validation curves exactly like this to decide when to stop.
