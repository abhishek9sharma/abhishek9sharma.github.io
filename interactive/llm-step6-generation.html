Generation: putting it all together

A trained model doesn't write a sentence in one shot. It predicts one token, appends it, and feeds the whole thing back in to predict the next — over and over. That loop is all "generation" is. Here's a small model that actually runs it live, so you can watch words appear.

01The Loop

Autoregression: predict, append, repeat

"Autoregressive" just means each new token depends on all the tokens before it — including the ones the model just generated. The model's own output becomes its next input:

Every token costs a full forward pass. That's why generating long text is slower than reading it, and why "tokens per second" is the headline speed metric for LLMs. Real systems cache past computation (the "KV cache") so they don't recompute everything each step — but the loop itself is exactly what you see above.
02Watch It Generate

A working model, generating live

This model was trained on a small set of simple sentences. Give it a starting word, set the temperature and sampling controls from Step 2, and press generate. Watch each token get chosen from a live probability distribution — the orange bar is the token it picked.

Next-token distribution

Press generate or step to see the model choose.

Lower the temperature to 0.2 for repetitive, "safe" text; raise it past 1.5 for chaos. Switch to greedy and it becomes fully deterministic — same start, same output every time.

Honesty about this demo: to run entirely in your browser with no downloads, this uses a small statistical (n-gram) model rather than a full neural Transformer — training a real one needs far more compute. But the generation loop, temperature, and sampling are identical to what GPT-class models do. The box that produces "probabilities for the next token" is the only part swapped; everything around it is the real thing.

Where this is heading

You now have the complete end-to-end picture. But generation has a performance problem: every new token re-reads all previous tokens' keys and values from scratch. The KV cache (Step 7) fixes that by storing them — and PagedAttention (Step 8) solves how to manage that memory efficiently in production.
